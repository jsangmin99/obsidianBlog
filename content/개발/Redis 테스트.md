- **성능 개선 효과**
    - Redis 캐싱을 도입하면 Read 속도가 최대 **20배** 향상됨.
    - Write는 데이터 동기화 로직이 필요하지만, 적절한 전략을 사용하면 성능 향상 가능.
- **비용 효율성 분석**
    - 단순 Redis 도입으로는 비용이 증가하지만, 트래픽 증가에 따른 DB 부하 감소로 전체 비용을 최적화할 수 있음.
    - DB의 스케일링 비용 vs Redis 도입 비용 비교 필요.
- **Redis 최적화 전략**
    - TTL 최적화: 필요 이상의 TTL 설정은 CPU 부하 증가
    - 캐시 미스 최소화: 적절한 키 설계로 캐시 히트율 증가
    - 데이터 일관성 유지: Write-Behind 또는 Lazy Write 기법 활용
